▓▓▓ O PROBLEMA REAL
────────────────────────────────────────
Pagamentos digitais não geram fatos 
confiáveis. Geram registros frágeis, 
controlados por terceiros.

Quando surge uma disputa, uma auditoria 
ou um conflito:
* O banco decide
* O gateway interpreta
* A plataforma pode apagar

O dinheiro passou, mas a verdade não ficou.

▓▓▓ O QUE FAZEMOS
────────────────────────────────────────
O FLOWPay transforma um pagamento em 
evidência verificável.

* PIX acontece no mundo real
* O valor é liquidado
* O evento é registrado
* A prova é soberana e auditável

Não é sobre pagar. É sobre não poder 
negar depois.

▓▓▓ NÃO É BANCO
────────────────────────────────────────
FlowPay não:
* Custodia fundos
* Cria saldo virtual
* Vende crédito
* Bloqueia conta

O sistema existe para registrar fatos, 
não para controlar pessoas.

▓▓▓ COMO FUNCIONA
────────────────────────────────────────
1. O cliente cria um link
2. O usuário paga via PIX
3. Valor é convertido e liquidado
4. Prova é imutável e pública

Sem prints. Sem planilhas. Sem confiar.
